# Content CMS — Optimise Digital

Internal agency operations platform for Optimise Digital: a Payload CMS/Next.js app that manages clients, proposals, contracts, audits, SEO/GSC data, Google Ads workflows, OptiMate agent approvals, finance, contractor tracking, and client-facing PIN/token portals.

## Stack confirmed from local config

- Next.js App Router (`next` 16) + React 19, wrapped with `@payloadcms/next` in `next.config.mjs`.
- Payload CMS 3 with SQLite/libSQL adapter in `src/payload.config.ts`; local default DB is `file:./content.db`, production-style URLs use Turso/libSQL with `DATABASE_AUTH_TOKEN`.
- Vercel Blob storage is enabled for the `media` collection only when `BLOB_READ_WRITE_TOKEN` is set.
- Rich text uses Payload Lexical plus `src/lib/markdown-paste-feature.ts`.
- Tests use Vitest 4 + jsdom with setup in `tests/setup.ts`; CI uses Node 22.

## Main structure

- `src/payload.config.ts` registers 73 collections, 7 globals, admin custom components, generated types output, DB adapter, and Vercel Blob plugin.
- `src/collections/` holds Payload collection configs for clients, proposals, contracts, SEO/GSC/audit records, Google Ads operations, agents, finance, contractors, content, notifications, and utility records.
- `src/globals/` holds singleton admin config: API cost rates, email templates, OptiMate settings, blog settings, Calendar/Sheets auth, and cron settings.
- `src/app/(payload)/` is the Payload admin surface; `src/app/(payload)/admin/importMap.js` contains the generated admin component import map.
- `src/app/(frontend)/` contains public pages and app-specific UI routes such as `proposals`, `audits`, `reports`, `partners`, `contracts`, `negative-keyword-build`, `client`, `schedule`, `contractor`, and `optimate-popout`.
- `src/app/(frontend)/api/` contains 76 API route groups, including audits, GSC, Gmail, GA4, Calendar, Sheets, Google Ads, negative keywords, contracts, invoice statements, agents, migrations, and diagnostics.
- `src/lib/` contains service clients and domain logic for GSC, Gmail, GA4, Sheets, Calendar, Google Ads helpers, contracts, invoices, dashboards, client pulse/revenue, screenshots/blob upload, agents, deck templates, and migrations.
- `src/migrations/` contains timestamped migration files; `src/lib/run-migrations.ts` is the bundled migration runner used by `/api/migrate`.
- `src/payload-types.ts` is generated by Payload and is committed.
- `tests/` mirrors app areas with unit/component/API/agent/app/e2e tests.

## Project workflows and constraints

- Schema changes are not auto-pushed in production: `src/payload.config.ts` only enables Payload `push` for `file:` database URLs unless `DISABLE_DB_PUSH=1`; deployed schema changes go through `POST /api/migrate` with `x-api-key: AUDIT_API_KEY`.
- CI (`.github/workflows/ci.yml`) runs `npm ci`, `npx tsc --noEmit`, `npm run build`, then on pushes to `main` waits for Vercel and calls `https://cms.optimisedigital.online/api/migrate`.
- Vercel cron routes are declared in `vercel.json` for GSC sync, dashboard prewarm, scheduled tasks, contract reminders, invoice statement sweep, contract trash sweep, match-type violations, Google Ads snapshots, and goal agents.
- `next.config.mjs` sets `X-Robots-Tag: noindex, nofollow` globally and redirects Payload `blog-prompts` collection screens to `/admin/blog/prompter`.
- `src/app/(payload)/admin/importMap.js` must retain the `@payloadcms/storage-vercel-blob/client#VercelBlobClientUploadHandler` import-map entry when regenerating admin imports.
- Public/client-facing areas commonly use PIN or token gates implemented in app routes and helpers such as `src/lib/pin-auth.ts`.
- OptiMate has TWO voice/dictation UIs, both fully wired and intentionally kept: `OptiMateVoice` (GPT Realtime WebRTC voice *call* via `src/lib/realtime/token-provider.ts` + the `realtime-session`/`realtime-tool`/`email-realtime-*` API routes) and `OptiMateTranscribe` (Apple/Web Speech dictation into the chat input). In `src/components/OptiMateChatCore.tsx` an `isMobileViewport` (`matchMedia('(max-width: 768px)')`) flag swaps them at render time only: mobile shows `OptiMateTranscribe`, desktop shows `OptiMateVoice` (still gated by `canUseVoice = isVoiceEnabled() && (mode === 'audit' || selectedAccountRefs.length > 0)`). The Realtime voice infrastructure is preserved on both platforms and NOT removed; it is only hidden on mobile by this render swap. To re-enable Realtime voice on mobile later, drop/relax the `isMobileViewport` gate at the two render sites (lines ~1365 and ~2218) — no backend, env, or file changes are needed. A clean future option is a single flag like `MOBILE_VOICE_MODE = 'transcribe' | 'realtime' | 'both'` instead of editing JSX.

## External services surfaced in code/config

- `PAYLOAD_SECRET` is required at Payload startup.
- `DATABASE_URL` / `DATABASE_AUTH_TOKEN` configure SQLite/libSQL/Turso.
- `BLOB_READ_WRITE_TOKEN` enables Vercel Blob media storage.
- `AUDIT_API_KEY` gates `/api/migrate`.
- Google integrations are present for GSC, Gmail, GA4, Calendar, and Sheets.
- Growth Tools/Scrapling-style HTTP integrations, SendGrid/Brevo email paths, Gemini/other LLM provider paths, and Vercel cron/API integrations are implemented under `src/app/(frontend)/api/` and `src/lib/`.
