# Feature Catalog — content-cms

Source-of-truth index that every test scenario references by **FEAT-ID**. One row per
feature across the platform (Payload CMS v3 + Next.js 16). This consolidates the codebase
survey described in `.gg/plans/platform-feature-test-swarm.md` **Phase 1**, reusing
`docs/api-endpoint-inventory.md` (239 CMS routes already inventoried) as raw material for the
API surfaces. Later phases (scenario design, the agent swarm, OptiMate/goal-agent validation)
reference features here by their stable FEAT-ID.

## Columns

| Column | Meaning |
|---|---|
| **id** | Stable FEAT-ID, `<PREFIX>-NNN` (e.g. `CLI-001`). Never renumber once assigned. |
| **name** | Short human name for the feature. |
| **domain** | The domain section the row belongs to (matches the FEAT-ID prefix). |
| **surface** | What kind of thing it is: `collection field` · `admin page` · `public route` · `API` · `agent tool`. |
| **purpose** | One sentence: what it does / why it exists. |
| **how-to-use** | The real user steps to exercise it. |
| **entry point** | URL, route path, or component path the user/test hits. |
| **external deps** | External services involved (Growth Tools, Gemini, GSC/GA4, Xero, Postmark/Brevo, Gmail, Vercel Blob, Google Sheets, …) or `—`. |
| **side-effect class** | One of READ / CMS-WRITE / EXTERNAL-SAFE / DANGER (see below). Gates what the test swarm may do. |

## Side-effect classes (Phase 2a)

| Class | Definition | Swarm policy |
|---|---|---|
| **READ** | No writes — dashboards, report views, list/status endpoints, OptiMate read tools. | May run freely. |
| **CMS-WRITE** | Writes only to the CMS DB — create proposal, save curation, draft NKL, process-step updates, memory writes, OAuth token storage, schema migrations. | May run against dedicated test records. |
| **EXTERNAL-SAFE** | External call with no client-visible effect — Growth Tools reads, GSC/GA4 queries, Gmail *draft* creation, Xero *list*, Gemini generation. | May run. |
| **DANGER** | Irreversible / client-visible — email sends, Google Ads budget push / campaign build / ad-copy deploy / negative push-live, Xero approve/send, contract send, Google Sheets write, calendar event creation, goal-agent live execute. | **Swarm must NOT trigger live.** Test via dry-run / preview / approval-staging or a sandbox account only. |
